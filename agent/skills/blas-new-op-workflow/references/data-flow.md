# 数据流说明

## 文档存储

所有开发中间文档存放在 `.claude/dev-docs/{operator_name}/`：

```
.claude/dev-docs/{operator_name}/
├── LOG.md                      # 开发日志
├── 2.0.1-开发环境.md
├── 1.1-参考资料清单.md
├── 1.2-需求分析.md
├── 1.3.A-开发方案设计.md
├── 1.3.B-测试方案设计.md
├── 1.4.A-开发方案评审.md
├── 1.4.B-测试方案评审.md
├── 2.1.2-汇合联调报告.md
├── 2.1.3-测试验收报告.md
├── 2.2.2-汇合联调报告.md
├── 2.2.3-测试验收报告.md
├── 3.1-代码检视报告.md
├── 3.2-性能报告.md
├── 4.2-代码检视报告.md
├── CP1.1.json ~ CP4.3.json
└── issues/
    └── issue_{YYYYMMDD}_{关键词}.md
```

## 阶段一：设计

```
用户需求
    │
    └─→ [1.1 资料准备] → 工作区目录 + LOG.md + 1.1-参考资料清单.md + CP1.1.json
            │
            └─→ ⛔ CP1.1（数据类型/目标芯片/编程模型/精度标准）

CP1.1结论 + 1.1-参考资料清单.md
    │
    └─→ [1.2 需求分析] → 1.2-需求分析.md + CP1.2.json
            │
            └─→ ⛔ CP1.2（用户审批：精度标准、约束确认）

1.2-需求分析.md
    ├─→ [1.3.A ⇄ 1.4.A 循环 ≤3] → 1.3.A + 1.4.A（开发侧：设计⇄评审）
    │
    └─→ [1.3.B ⇄ 1.4.B 循环 ≤3] → 1.3.B + 1.4.B（测试侧：设计⇄评审）

1.4.A + 1.4.B 双方通过
    └─→ ⚪ 1.4（任一方3次失败→CP问卷）
```

## 阶段二：开发

```
[2.0.1 环境准备] → 2.0.1-开发环境.md + git 分支 (算子名)
    │ （不通过→AskUserQuestion）
    │
1.3.A-开发方案设计.md → [2.1.1.A 算子开发] → 算子代码    ┐
1.3.B-测试方案设计.md → [2.1.1.B 测试开发] → 用例代码    ┤ 并行
                                                          │
算子代码 + 用例代码                                        │
    └─→ [2.1.2 汇合联调] → [2.1.3 测试验收] → ⚪ 2.1     │
                                                          │
（迭代二结构相同）                                         │
```

## 阶段三：验收

```
全部代码 → [3.1 代码检视] → 3.1-代码检视报告.md    ┐
1.2 + 1.3.A → [3.2 性能验收] → 3.2-性能报告.md      ┤ 并行
                                                     │
3.1 + 3.2 → ⚪ 3.2                                   │
```

## 阶段四：上库

```
全部代码 + 设计文档
    └─→ [4.1 编写文档] → README.md
    └─→ [4.2 代码检视] → 4.2-代码检视报告.md
    └─→ [4.3 开发总结] → CP4.3.json
            └─→ ⛔ CP4.3（AskUserQuestion → squash commit）
```

# AscendC 编码规范

| 编号 | 规则 | 说明 |
|------|------|------|
| R1 | 禁止逐元素操作 | 禁止 SetValue/GetValue、逐元素 DataCopy → [R1-禁止逐元素操作.md](references/R1-禁止逐元素操作.md) |
| R2 | 动态获取 CoreNum | 禁止硬编码核数，使用 aclrtGetDeviceInfo → [R2-动态获取CoreNum.md](references/R2-动态获取CoreNum.md) |
| R3 | TPipe 禁止作为成员变量 | TPipe 应在 Kernel 入口创建 → [R3-TPipe禁止成员变量.md](references/R3-TPipe禁止成员变量.md) |
| R4 | TilingData 禁止使用数组 | 不可用数组为每个核分配独立地址，应在 TilingData 中记载核间间隔，由 Kernel 根据 blockIdx 自行计算 → [R4-TilingData禁止数组.md](references/R4-TilingData禁止数组.md) |
| R5 | 圈复杂度 ≤ 20 | 函数的 Cyclomatic Complexity 不超过 20，超过需拆分子函数 |
| R6 | 函数深度 ≤ 5 | 最大嵌套层级不超过 5，超过需提取内层循环/分支为独立函数 |
| R7 | 函数行数 ≤ 50 | NBNC（非空非注释行）不超过 50，超过需拆分为多个函数 |
| R8 | 除零防御 | 除法/取模运算的被除数必须校验非零，特别是来自外部输入的变量（如 coreNum） |
| R9 | 许可证头 | 所有源码文件必须包含标准许可证头（CSV 文件除外） |
| R10 | 禁止 extern 引用 | 禁止在 Kernel/Host 代码中使用 `extern "C"` 声明外部函数接口，应通过头文件 include 引入 |

# MR 安全编码规则

## 使用方式

- **开发阶段**：编码时对照规则自查，避免引入违规
- **MR 前检查**：提交前逐条扫描代码，修正违规项
- **检视阶段**：reviewer 对照规则审核代码结构合规性

## 规则分类

### 1. 预处理器规则（G.PRE）

| 编号 | 规则 | 级别 | 要求 |
|------|------|------|------|
| G.PRE.05 | `#else`/`#elif`/`#endif` 必须与对应的 `#if`/`#ifdef`/`#ifndef` 在同一文件中 | 严重 | 禁止跨文件分段预处理 |
| G.PRE.07 | 宏名称不能与关键字相同 | 严重 | 使用 `#define` 时避开 C/C++ 关键字 |

### 2. 头文件/包含规则（G.INC）

| 编号 | 规则 | 级别 | 要求 |
|------|------|------|------|
| G.INC.05 | 禁止在 `extern "C"` 中 `#include` 头文件 | 严重 | `extern "C"` 块内只能有函数/变量声明 |
| G.INC.08 | 禁止在 `#include` 之前使用 `using` 导入 namespace | 严重 | `#include` 必须位于所有 `using` 之前 |

### 3. 表达式与声明规则（G.EXP）

| 编号 | 规则 | 级别 | 要求 |
|------|------|------|------|
| G.EXP.26 | 整型表达式比较或赋值为更大类型前，必须用该更大类型求值 | 一般 | 先转型再运算，避免溢出 |
| G.EXP.33 | 含变量自增/自减的表达式中禁止再次引用该变量 | 严重 | 如 `a = b[i++] + b[i]` 禁止 |

### 4. 控制流规则（G.CTL）

| 编号 | 规则 | 级别 | 要求 |
|------|------|------|------|
| G.CTL.03 | 循环必须安全退出 | 建议 | 禁止死循环，循环变量必须可控 |

### 5. 断言规则（G.AST）

| 编号 | 规则 | 级别 | 要求 |
|------|------|------|------|
| G.AST.03 | 禁止用断言检测运行期间可能发生的错误 | 一般 | 运行时错误使用错误处理代码（返回值校验/异常），非 assert |

### 6. 安全函数规则（G.FUU）

| 编号 | 规则 | 级别 | 要求 |
|------|------|------|------|
| G.FUU.09 | 禁止使用 `realloc()` | 严重 | 使用安全内存管理替代 |
| G.FUU.10 | 禁止使用 `alloca()` 申请栈上内存 | 严重 | 栈上动态分配易导致栈溢出 |
| G.FUU.11 | 必须检查安全函数返回值并正确处理 | 一般 | 所有安全函数的返回值必须校验 |
| G.FUU.12 | 正确设置安全函数中的 `destMax` 和 `count` 参数 | 严重 | 确保 destMax 反映真实缓冲区大小 |
| G.FUU.13 | 禁止封装安全函数 | 严重 | 禁止对安全函数再封装 |
| G.FUU.14 | 禁止用宏重命名安全函数 | 一般 | 保持安全函数原名 |
| G.FUU.15 | 只能使用安全函数库中的安全函数 | 严重 | 禁止使用不安全替代品 |

### 7. 数组规则（G.ARR）

| 编号 | 规则 | 级别 | 要求 |
|------|------|------|------|
| G.ARR.03 | 禁止通过对指针变量进行 sizeof 操作来获取数组大小 | 一般 | `sizeof(ptr)` 返回指针大小非数组大小 |

### 8. 内存规则（G.MEM）

| 编号 | 规则 | 级别 | 要求 |
|------|------|------|------|
| G.MEM.04 | 内存中敏感信息使用完毕后立即清零 | 严重 | 使用安全 memset 清除 |

### 9. 标准库规则（G.STD）

| 编号 | 规则 | 级别 | 要求 |
|------|------|------|------|
| G.STD.05 | 字符串操作缓冲区必须有足够空间容纳数据和结束符 | 严重 | 确保 null 结尾 |
| G.STD.07 | 禁止使用 `std::string` 存储敏感信息 | 严重 | 敏感信息使用安全内存管理 |
| G.STD.13 | 调用格式化输入/输出函数时使用有效的格式字符串 | 严重 | 禁止格式字符串与参数类型不匹配 |
| G.STD.15 | 禁止外部可控数据作为进程启动/dlopen/LoadLibrary 参数 | 致命 | 防止注入攻击 |
| G.STD.16 | 禁用程序退出/线程退出函数和 `atexit` | 严重 | 禁止 `exit`/`_exit`/`abort`/`atexit` |
| G.STD.17 | 禁止调用 `kill`/`TerminateProcess` 直接终止其他进程 | 严重 | 防止资源泄漏 |
| G.STD.18 | 使用库函数时避免竞争条件 | 一般 | 多线程环境下使用线程安全版本 |

### 10. 安全与合规规则（G.OTH）

| 编号 | 规则 | 级别 | 要求 |
|------|------|------|------|
| G.OTH.03 | 禁用 `rand` 产生用于安全用途的伪随机数 | 一般 | 安全场景使用密码学安全随机数 |
| G.OTH.05 | 禁止代码中包含公网地址 | 严重 | 代码中不得包含公网 IP/URL |

### 11. 资源规则（G.RES）

| 编号 | 规则 | 级别 | 要求 |
|------|------|------|------|
| G.RES.02 | 内存申请前必须对申请内存大小进行合法性校验 | 严重 | 校验大小 > 0 且不超过上限 |

### 12. 文件完整性规则（OAT）

| 编号 | 规则 | 级别 | 要求 |
|------|------|------|------|
| OAT.1 | 不允许包含二进制文件 | 严重 | 仓库中禁止提交二进制/.o/.so 等 |

### 13. 代码质量规则

| 编号 | 规则 | 级别 | 要求 |
|------|------|------|------|
| CQ.04 | 超大源文件：源文件总行数有上限要求 | 一般 | 超大的源文件应拆分为多个模块 |
| CQ.05 | 超大头文件：头文件内容有上限要求 | 一般 | 头文件应精简，避免膨胀 |
| CQ.06 | 超大目录：目录下文件数有上限要求 | 一般 | 文件过多的目录应分层组织 |
| CQ.07 | 重复代码：禁止重复代码块 | 一般 | 相同逻辑应提取为公共函数 |
| CQ.08 | 重复文件：禁止存在内容相同的文件 | 一般 | 发现重复文件应合并 |
| CQ.09 | 冗余代码：禁止未使用的代码 | 一般 | 包括未使用的 include/变量/函数/死代码等 |
| CQ.10 | 不安全函数：禁止使用不安全函数 | 一般 | 使用安全函数版本替代 |
| CQ.11 | 告警抑制：禁止无理由抑制编译告警 | 一般 | 告警应修复而非抑制 |

### 14. IPSI 算法安全

| 编号 | 规则 | 级别 | 要求 |
|------|------|------|------|
| CIP.01 | 不安全 IPSI 算法：禁止使用不安全的密码算法 | 致命 | 使用合规算法替代 |

## 优化检查流程

### 步骤 1：文件级检查
- [ ] 无二进制文件混入（OAT.1）
- [ ] 所有源码文件含许可证头（R9）
- [ ] 无公网地址（G.OTH.05）
- [ ] 无重复文件（CQ.08）
- [ ] 目录文件数在合理范围（CQ.06）

### 步骤 2：头文件检查
- [ ] 无 `extern "C"` 包含头文件（G.INC.05）
- [ ] `#include` 在 `using` 之前（G.INC.08）
- [ ] 无 extern 函数/变量声明（R10）
- [ ] 头文件大小在合理范围（CQ.05）

### 步骤 3：函数级检查
- [ ] 圈复杂度 ≤ 20（R5）
- [ ] 嵌套深度 ≤ 5（R6）
- [ ] NBNC ≤ 50（R7）
- [ ] 函数功能单一，无重复逻辑（CQ.07）
- [ ] 循环有安全退出条件（G.CTL.03）
- [ ] 断言不用于运行时错误检测（G.AST.03）

### 步骤 4：表达式与运算检查
- [ ] 除法/取模前校验被除数非零（R8）
- [ ] 整型表达式比较前已转型为更大类型（G.EXP.26）
- [ ] 自增/自减表达式无重复引用（G.EXP.33）
- [ ] 无 `sizeof(ptr)` 代替 `sizeof(array)`（G.ARR.03）

### 步骤 5：安全函数检查
- [ ] 无不安全函数调用（realloc/alloca）（G.FUU.09/10）
- [ ] 安全函数返回值已检查（G.FUU.11）
- [ ] `destMax` 参数正确设置（G.FUU.12）
- [ ] 未封装/重命名安全函数（G.FUU.13/14）
- [ ] 使用华为安全函数库（G.FUU.15）

### 步骤 6：内存与资源检查
- [ ] 内存申请前校验大小合法性（G.RES.02）
- [ ] 敏感信息使用后已清零（G.MEM.04）
- [ ] 字符串缓冲区足够容纳数据和结束符（G.STD.05）
- [ ] 不使用 `std::string` 存敏感信息（G.STD.07）

### 步骤 7：标准库与安全检查
- [ ] 格式化字符串与参数类型匹配（G.STD.13）
- [ ] 不使用外部可控数据启动进程（G.STD.15）
- [ ] 不使用 exit/atexit/kill 等危险函数（G.STD.16/17）
- [ ] 多线程场景使用线程安全版本（G.STD.18）
- [ ] 不使用 `rand` 做安全用途（G.OTH.03）
- [ ] 不使用不安全 IPSI 算法（CIP.01）

### 步骤 8：冗余与告警检查
- [ ] 无未使用的 include/变量/函数（CQ.09）
- [ ] 无死代码（CQ.09）
- [ ] 无重复代码块（CQ.07）
- [ ] 编译告警已修复而非抑制（CQ.11）

## 优化修复建议

| 违规类型 | 修复方法 |
|---------|---------|
| 宏名与关键字冲突 | 重命名宏，添加前缀或作用域标识 |
| 跨文件预处理分段 | 合入同一文件，或重构条件编译逻辑 |
| extern 声明引用 | 替换为头文件 include |
| 除零风险 | 在运算前添加非零校验 |
| 自增/自减重复引用 | 拆分为独立语句 |
| 断言检测运行时错误 | 替换为 if + 错误处理 |
| 安全函数未检查返回值 | 添加返回值校验和错误处理 |
| destMax 错误 | 使用 `sizeof` 或明确大小常量 |
| 圈复杂度超标 | 提取条件块为独立函数 |
| 嵌套深度超标 | 提取内层为独立函数 |
| 函数行数超标 | 按功能拆分为多个函数 |
| 重复代码 | 提取为公共函数或模板 |
| 冗余代码（未使用） | 删除 |
| 不安全函数 | 替换为安全函数版本 |
| 告警抑制 | 修复告警的根本原因 |
